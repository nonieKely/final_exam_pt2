Introduction
Pour cet examen, vous devez créer une première version du site web de l'hôtel “Le Lotus Bungalow” : https://www.facebook.com/lelotusfoulpointe/. 

Pour cela, un des gérants, Herilala, met à votre disposition les ressources ci-joint: https://drive.google.com/drive/folders/12hwhsJlx4rjgAvS2SP9E0pIsHSX1cmlq?usp=sharing. Il est également ouvert au fait que vous utilisez d’autres photos de l'hôtel en question si vous en trouvez, il n’y a également pas de soucis si vous utilisez seulement une partie des ressources proposées, ou si vous inventez des textes fictifs (mais réalistes, pas de lorem ipsum).


Indications
Pour faire ce fameux site, voici quelques indications :

Pas de template cette année, vous avez carte blanche: vous devez tout simplement coder en binôme sur un repo Git. Vous devez vous départager les pages et dire dans le mail qui a fait quoi (commit history à l’appui dans votre repo).
Si vous ne trouvez pas de partenaire, on peut vous assigner un d’office, ou si, plus personne n’est disponible, vous pourrez coder seul.
Notes de votre contribution: /15, notes de la synergie[1] de vos designs sur le même site : /5 (Astuce : un framework peut vous forcer à coder avec le même style);
Votre site doit être responsive dans son entièreté et doit passer sans la moindre erreur le validateur HTML du World wide web consortium https://validator.w3.org/ (reférence: Vidéo explicative dans HEI Tribu);
Les conventional commits sont obligatoires. Veuillez inviter en tant que collaborateur les mails suivants : tokyramarozaka@gmail.com, et hei.jean.3@gmail.com ;
L’objet du mail pour envoyer le repo doit être : PROJET WEB1 FINAL 2023-2024 <std-étudiant-1> <std-étudiant-2> (plus besoin d’envoyer un e-mail le jour J);
Vous pouvez rajouter autant de pages et de sections que vous le souhaitez, mais nous estimons que 4 sections ou 4 pages devraient suffire pour cette première version;
Vous pouvez utiliser autant de bibliothèque que vous le souhaitez (Font awesome, Tailwind, Material, Bootstrap, Photostack …);
Vous devez cependant fournir la liste des technologies utilisées dans le mail (ex : nous avons utilisé AOS, React, Vue, ChartJs etc.), s’il y a des instructions en plus pour lancer l’application : les inclure dans le readme.
Toute technologie externe à du HTML, CSS, JS non listée dans le mail ou le readme représente une faute de communication, et sera sanctionné d’un malus;
Vous pouvez utiliser https://www.remove.bg/fr pour enlever le fond de certaines photos si besoin est.
Modalité d’envoi : envoyez par e-mail le lien du repo à toky@hei.school avec en copie hei.jean.3@gmail.com.
Deadline : Vendredi 7 Juin 2024 (inclus), avant minuit
D'autres questions ? demandez !
